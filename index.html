1. Como criar um mapa e marcadores:

Em um arquivo html, com a biblioteca Leaflet já adicionada, você deve abrir um script e nele colocar o seguinte código (esse código também pode ser pego no site do Leaflet):

var map = L.map('map').setView([0, 0], 1);

L.tileLayer( 'https://api.maptiler.com/maps/streets-v2/{z}/{x}/{y}.png?key=0xLQbt4WJeeuwW3p39Hy',

{attribution: '© MapTiler © OpenStreetMap contributors' }).addTo(map);

var marker = L.marker([51.5, -0.09]).addTo(map);

Durante o codigo pode ser visto a utilização de novas "tags", uma breve expllisicação sobre o use delas:

  • Var Map

    É a declaração de uma variável que armazena uma instância de algum elemento do mapa.

  • L.map

    • L. : é um atalho para acessar toda as funções e classes do Leaflet

    • L.map : é um exemplo de uma função

  • L.tileLayer e attribution

    • L.tileLayer : é a função para criar a camada de tiles

    • attribution : É o credito dos criadores do mapa

  • setView([0, 0], 1)

    ([lat,lng], zoom), lat: latitude; lng: longitude;

O que é escala?

A escala é uma representação específica usada em mapas para indicar a relação entre as distâncias no mapa e as distâncias correspondentes no terreno real.

Ela pode ser expressa de três maneiras:

  • Numericamente (como 1:100.000,indicando que 1 unidade no mapa equivale a 100.000 unidades no mundo real)
  • Linha de escala (uma barra dividida em segmentos marcados com distâncias reais)
  • Verbalmente (como "Um centímetro equivale a um quilômetro").

O que é o GeoJSON?

GeoJSON é uma iniciativa open-source desenvolvida para organizar as estruturas geográficas do nosso planeta em formato de código. Ele é baseado no JSON (JavaScript Object Notation), proporcionando uma maneira eficiente de redivsentar e compartilhar dados geoespaciais.

Um exemplo de uso do GeoJSON é para redivsentar dados geográficos em mapas interativos da web. Por exemplo, você pode usar o GeoJSON para marcar locais específicos, desenhar áreas geográficas, como estados ou países. Esses dados podem ser então incorporados em aplicativos de mapeamento, como o Google Maps ou o Leaflet, para criar visualizações interativas e informativas.

7. Como colocar GeoJSON em meu mapa?

Voce pode importar poligonos e marcacoes de Json, atraves do GeoJson.io, podendo criar arquivos em JS e geoJson,para colocar em seu mapa. Escolha a exportação como .js assim

fetch('Data/linha.geojson')
.then(response => {
  if (!response.ok) {
  throw new Error('A network diz que tem algo   errado!');
  }
  return response.json();
})

.then(data => {
  L.geoJSON(data, {
  style: function () {
   return { color: 'black' };
  }
}).bindPopup(function () {
  return "Rota Feita";
  }).addTo(map);
})

.catch(error => {
console.error('Houve um problema com a requisição Fetch:', error);
});

const geojsonFiles = [
fetch('Data/linha.geojson'),

Promise.all(geojsonFiles)
.then(responses => {
  // Verifica se todas as respostas foram bem-sucedidas

    if (!response.ok) {
      throw new Error('A network diz que tem algo errado!');
    }
  });

  // Converte todas as respostas em JSON
  return Promise.all(responses.map(response => response.json()));

  });
})

.catch(error => {
console.error('Houve um problema com a requisição Fetch:', error);
});

Durante o codigo pode ser visto a utilização de novas "tags", uma breve expllisicação sobre o use delas:

  • fetch('Data/linha.geojson')

    A função fetch é uma API do JavaScript que permite fazer requisições assíncronas a recursos de rede, como arquivos ou APIs.

    Neste caso: Está sendo feita uma requisição para buscar o arquivo linha.geojson que está localizado na pasta Data. A função retornará uma Promise que resolve a resposta da requisição.

  • Promise

    é um objeto que representa a eventual conclusão (ou falha) de uma operação assíncrona e seu valor resultante.

    Ela tem 3 status

      Pending (Pendente): O estado inicial, aguardando que a operação assíncrona seja concluída.

      Fulfilled (Cumprida): A operação foi concluída com sucesso e a promessa possui um resultado.

      Rejected (Rejeitada): A operação falhou e a promessa possui um motivo para a falha.
  • Operação Assíncrona

    É uma operação que é iniciada e permite que outras operações continuem em execução enquanto aguarda a conclusão

  • .then(response => { ... })

  • É usado para tratar a resposta da Promise retornada pelo fetch.

  • if (!response.ok) { throw new Error('A network diz que tem algo errado!'); }

    • !response.ok : procura se o status do response é diferente de ok

    • throw new Error : Caso o status for diferente de ok, é para "jogar um erro" no console

  • return response.json();

    É usado para ler o corpo da resposta e convertê-lo de GeoJSON para um objeto JavaScript.

  • .catch(error => { console.error('Houve um problema com a requisição Fetch:', error); });

    Se houver um erro (como uma falha na rede ou um erro ao processar a resposta), uma mensagem de erro será exibida no console

  • Promise.all(...)

    É um método que aceita um array de Promises e retorna uma única Promise

  • responses.forEach(response => { if (!response.ok) { throw new Error('A network diz que tem algo errado!'); } });

    Verifica o status de cada resposta. Se response.ok for false para qualquer uma das respostas, da erro.

  • return Promise.all(responses.map(response => response.json()));

    Cria um novo array de Promises, cada uma delas convertendo uma resposta do geoJSON.

O que é CSV?

CSV é a sigla para Comma Separated Values (em português, valores separados por vírgulas). Um arquivo CSV é um arquivo de texto simples que armazena informações de planilhas e tabelas. O conteúdo geralmente é uma tabela de texto, números ou datas.

Em geral, a primeira linha em um arquivo CSV contém os rótulos das colunas da tabela. Cada uma das linhas subsequentes redivsenta uma linha da tabela. Vírgulas separam cada célula na linha, que é a razão para o nome do formato.

Country, Name, lat, lng, Altitude
United States, New York City, 40.7142691, -74.0059738, 2.0
United States, Los Angeles, 34.0522342, -118.2436829, 115.0

Durante o codigo pode ser visto a utilização de novas "tags", uma breve expllisicação sobre o use delas:

  • Continue
    Uma declaração JavaScript utilizada dentro de loops para pular a posição atual e continuar com a próxima posição do loop
  • row.lat e row.lng
    São acessos aos campos "lat" e "lng" do objeto "row", que representa uma linha de dados do CSV analisado.
  • Papa.parse()
    PapaParse é uma biblioteca JavaScript usada para analisar dados CSV. Ela converte dados CSV em arrays ou objetos JavaScript.
  • Parâmetros de configurações usados de um Papa.parse()
    • header
      Um parâmetro que indica se a primeira linha do CSV contém cabeçalhos de coluna. Se definido como true, a primeira linha será interpretada como o cabeçalho da coluna.
    • dynamicTyping
      Um parâmetro que controla se os valores das células devem ser automaticamente convertidos para os tipos de dados apropriados (por exemplo, números, booleanos) com base em seu formato. Se definido como true, Papa.parse() tentará colocar automaticamente os tipos de dados das células.
    • delimiter
      Um parâmetro que especifica o caractere delimitador utilizado para separar os campos no CSV. No exemplo, o delimitador é definido como "|".
  • NaN
    NaN é uma abreviação de "Not a Number", é um valor especial em JavaScript que representa um valor que não é um número válido.
