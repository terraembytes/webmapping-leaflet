5. Escala

O que é escala?

A escala é um tipo específico de escala usada em mapas para representar a relação entre as distâncias no mapa e as distâncias correspondentes no terreno real.

Ela pode ser expressa de três maneiras: numericamente (como 1:100.000, indicando que 1 unidade no mapa equivale a 100.000 unidades no mundo real), por meio de uma linha de escala (uma barra dividida em segmentos marcados com distâncias reais) ou verbalmente (como "Um centímetro equivale a um quilômetro").

Como colocar escala? Ou como colocar Marca D'agua

Para adicionar a escala e uma marca d'água ao seu mapa, siga estas etapas simples:

7. GeoJSON

O que é o GeoJSON?

GeoJSON é uma iniciativa open-source desenvolvida para organizar as estruturas geográficas do nosso planeta em formato de código. Ele é baseado no JSON (JavaScript Object Notation), proporcionando uma maneira eficiente de representar e compartilhar dados geoespaciais.

Um exemplo de uso do GeoJSON é para representar dados geográficos em mapas interativos da web. Por exemplo, você pode usar o GeoJSON para marcar locais específicos, desenhar áreas geográficas, como estados ou países. Esses dados podem ser então incorporados em aplicativos de mapeamento, como o Google Maps ou o Leaflet, para criar visualizações interativas e informativas.

10. CSV

O que é o CSV?

CSV é a sigla para Comma Separated Values (em português, valores separados por vírgulas). Um arquivo CSV é um arquivo de texto simples que armazena informações de planilhas e tabelas. O conteúdo geralmente é uma tabela de texto, números ou datas.

Em geral, a primeira linha em um arquivo CSV contém os rótulos das colunas da tabela. Cada uma das linhas subsequentes representa uma linha da tabela. Vírgulas separam cada célula na linha, que é a razão para o nome do formato.

                Country, Name, lat, lng, Altitude
                United States, New York City, 40.7142691, -74.0059738, 2.0
                United States, Los Angeles, 34.0522342, -118.2436829, 115.0

Como colocar dados de um CSV no mapa?

Para realizar tal façanha será preciso importar 2 bibliotecas "novas" o JQuery e PapaParse, ambas seram utilizadas para ler determinado CSV, de sua escolha. 

                $.get('Data/datacsv.csv', function (csvString) {
                    var data = Papa.parse(csvString, { header: true, dynamicTyping: true, delimiter: "|" }).data;
                    for (var i = 0; i < data.length; i++) {
                        var row  = data[i];
                
                        if (isNaN(row.lat) || isNaN(row.lng)) {
                            console.log("Coordenadas inválidas para o marcador:", row);
                            continue;
                        }
                
                        if (row.lat === null || row.lng === null) {
                            console.log("Coordenadas ausentes para o marcador:", row);
                            continue;
                        }
                
                        var marker = L.marker([row.lat, row.lng], {
                            opacity: 1
                        }).bindPopup(row.Name);
                
                        marker.addTo(map);
                    }
                });

Durante o codigo pode ser visto a utilização de novas "tags", uma breve expllisicação sobre o use delas:

  • Continue
    Uma declaração JavaScript utilizada dentro de loops para pular a posição atual e continuar com a próxima posição do loop
  • $.get()
    É uma função do jQuery utilizada para realizar uma requisição AJAX para carregar dados de um servidor. Neste caso, ela é usada para carregar o arquivo CSV contendo os dados dos marcadores.
  • row.lat e row.lng
    São acessos aos campos "lat" e "lng" do objeto "row", que representa uma linha de dados do CSV analisado.
  • Papa.parse()
    PapaParse é uma biblioteca JavaScript usada para analisar dados CSV. Ela converte dados CSV em arrays ou objetos JavaScript.
  • Parâmetros de configurações usados de um Papa.parse()
    • header
      Um parâmetro que indica se a primeira linha do CSV contém cabeçalhos de coluna. Se definido como true, a primeira linha será interpretada como o cabeçalho da coluna.
    • dynamicTyping
      Um parâmetro que controla se os valores das células devem ser automaticamente convertidos para os tipos de dados apropriados (por exemplo, números, booleanos) com base em seu formato. Se definido como true, Papa.parse() tentará colocar automaticamente os tipos de dados das células.
    • delimiter:
      Um parâmetro que especifica o caractere delimitador utilizado para separar os campos no CSV. No exemplo, o delimitador é definido como "|".
  • NaN
    NaN é uma abreviação de "Not a Number", é um valor especial em JavaScript que representa um valor que não é um número válido.
